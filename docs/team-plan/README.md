## Working Rule

- Only one sprint is active at a time.
- Right now, only `sprint-01-current.md` is active.
- Sprint 02 to Sprint 06 are planning references, not permission to start early.
- When a teammate finishes a task, review it first.
- Only after validation should work be marked complete or merged forward.
- New sprint work starts only after the current sprint is accepted.

## Source Rule

- The sprint files follow `docs/SOFTWARE_CONCEPTION.md`.
- The sprint sequence follows `.team-review/roadmap-3-month.md`.
- If there is a conflict, the software conception is the product source of truth.
