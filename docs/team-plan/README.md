# Team Plan

This directory is the public team planning area.

## Active Files

- `sprint-01-current.md`: active 2-week sprint

## Working Rule

- Only `sprint-01-current.md` is active now.
- When a teammate finishes a task, review it first.
- Only after validation should work be marked complete or moved forward.
- New tasks are assigned only after the reviewer validates completed work.

## Fairness Rule

- Work should be equal by effort, not only by task count.
- No teammate should receive extra hidden work outside the sprint file.
- Do not duplicate the same ownership across two people.
- Each feature should have one primary owner.
- If one task is clearly heavier than another, rebalance before assigning the sprint.
